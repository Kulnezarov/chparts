# ✅ Pre-Release Checklist для CHPARTS

## 🔍 Quality Assurance

### Ошибки & Обработка
- [ ] Открыть несуществующую страницу `/nonexistent` → должна показаться красивая 404
- [ ] Вызвать ошибку в консоли (откройте DevTools и `throw new Error()`) → должна показаться error.tsx страница
- [ ] Проверить, что ошибки имеют кнопки "Обновить" и "На главную"
- [ ] **Edge case**: Отключить интернет → попробовать добавить товар → должно показаться сообщение об ошибке

### Корзина & Checkout
- [ ] Добавить несколько товаров в корзину
- [ ] Проверить, что сумма считается правильно
- [ ] Попробовать отправить пустую форму → все поля должны подсветиться красным
- [ ] Напечатать только 5 цифр в телефоне → ошибка "Введите номер полностью"
- [ ] Выбрать "Доставка по Шымкенту" → поле адреса должно появиться и скрыться при выборе другого варианта
- [ ] Выбрать "Другой город/почта" → должны появиться поля города и деталей
- [ ] При выборе "Доставка" или "Почта" → опция "Наличные" должна исчезнуть
- [ ] Заполнить форму корректно и отправить → должен появиться Success экран с номером резерва
- [ ] Скопировать номер резерва (кнопка с иконкой скопировать) → должно показаться "Скопировано"
- [ ] Нажать "Продолжить покупки" → корзина должна очиститься
- [ ] Написать комментарий с максимальной длиной → не должно обрезаться

### Delivery Types
- [ ] **Pickup**: показывается адрес "Шымкент, Исмаил ата 403"
- [ ] **City**: показывается поле для адреса + сообщение "уточним у менеджера"
- [ ] **Post**: показываются поля для города и деталей отправки

### Payment Types
- [ ] **Pickup**: показываются обе опции (карта + наличные)
- [ ] **City/Post**: показывается только карта с сообщением "менеджер пришлёт реквизиты"

### Товары в каталоге
- [ ] Открыть товар и нажать "В корзину"
- [ ] Удалить товар из корзины
- [ ] Увеличить количество товара (кнопка +)
- [ ] Уменьшить количество товара (кнопка -)
- [ ] Проверить валидацию (не больше доступного количества)

## 🧪 Edge Cases

### Validation
- [ ] Имя с 1 буквой → ошибка
- [ ] Имя с 200 буквами → должно приниматься (но обрезаться в backend)
- [ ] Телефон с буквами → должно игнорироваться
- [ ] Адрес с спецсимволами (ул. №/-, КВ.) → должно приниматься
- [ ] Очень длинный комментарий (500+ символов) → должно быть ограничено

### Network
- [ ] Медленное соединение (throttle в DevTools) → должны показаться правильные лоадинги
- [ ] Отправить форму → прервать соединение посередине → должно показаться сообщение об ошибке
- [ ] Rate limit (429): если отправить несколько запросов подряд → должно показаться специальное сообщение

### Состояния
- [ ] Кнопка "Оформить" в процессе отправки → должна быть disabled
- [ ] При пустой корзине → кнопка "Оформить" должна быть disabled
- [ ] После успешного заказа → форма должна очиститься
