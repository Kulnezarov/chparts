### 4. **Обработка API ошибок** (`lib/apiErrorHandler.ts`)

```typescript
try {
  await createPublicOrder(data);
} catch (err) {
  const msg = ApiErrorHandler.getErrorMessage(
    err,
    "Ошибка по умолчанию"
  );
}
```

Автоматически распознает:
- 429 → "Слишком много попыток"
- 400 → "Проверьте данные"
- 500+ → "Ошибка сервера"
- Network error → "Проверьте интернет"

## 🔧 Детали реализации

### Checkout Flow (Логика оформления)

```
1. Пользователь выбирает способ доставки (pickup/city/post)
   ↓
2. Система автоматически обновляет поле оплаты
   - pickup: карта ИЛИ наличные
   - city/post: только карта
   ↓
3. Пользователь заполняет контакты
   ↓
4. BEFORE submit → валидация на клиенте
   ↓
5. Если ошибки → показываем Alert с деталями
   ↓
6. Если OK → отправляем на backend
   ↓
7. Backend может вернуть 429 (Rate limit) → специальное сообщение
   ↓
8. На успех → очищаем корзину + показываем Success screen
```

### Event Handlers

```typescript
// Очистка ошибок при вводе
onChange={(e) => {
  setName(e.target.value);
  if (fieldErrors.name) 
    setFieldErrors(x => ({ ...x, name: "" }));
}}

// Автоматический выбор оплаты
useEffect(() => {
  if (deliveryType !== "pickup") 
    setPaymentType("card");
}, [deliveryType]);
```
